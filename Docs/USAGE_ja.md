# Folder Cleaner ユーザーガイド (USAGE)

**Folder Cleaner** は、指定したフォルダ内にあるテクスチャアセットの中から、プロジェクト内の特定のフォルダ（参照元フォルダ）にあるアセットから使用（参照）されていないテクスチャを検出・削除する Unity エディタ拡張ツールです。

---

## 1. 起動方法

Unity エディタの上部メニューバーから、以下を選択します。
- **dennokoworks** ＞ **Folder Cleaner**

起動すると、フローティングデザインの専用ウィンドウが表示されます。

---

## 2. 設定項目

ウィンドウは「フォルダ設定」「オプション」「結果」の3つのセクションで構成されています。

### フォルダ設定
| 設定項目 | 説明 |
| :--- | :--- |
| **テクスチャ検索フォルダ** | 整理（削除候補を検索）したいテクスチャが保存されているフォルダを指定します。※サブフォルダ内も検索対象になります。 |
| **参照元フォルダ** | テクスチャが使用されている可能性があるフォルダ（例: `Assets/Materials` や `Assets/Prefabs` など）を指定します。「＋ 参照元フォルダを追加」ボタンで複数指定可能です。 |
| **除外サブフォルダ** | 参照元の検索から除外したいフォルダを指定します。「＋ 除外サブフォルダを追加」ボタンで複数指定可能です。 |

> [!NOTE] **自動除外機能について**
> 「テクスチャ検索フォルダ」が「参照元フォルダ」のいずれかのサブフォルダである場合、自分自身の中にあるアセットからの参照によって「使用中」と判定されてしまう（自己参照）のを防ぐため、自動的に「除外サブフォルダ」へ追加されます。

### オプション
- **削除方法**:
  - **ゴミ箱へ移動**: 削除したテクスチャは OS のゴミ箱へ移動します（後から復元可能です。推奨）。
  - **完全に削除**: 削除したテクスチャはプロジェクトから完全に削除されます（復元できません）。

---

## 3. 使用手順

### ステップ 1: フォルダの指定
1. 「テクスチャ検索フォルダ」に対象フォルダをドラッグ＆ドロップまたはアセットピッカーから指定します。
2. 「参照元フォルダ」に検索の起点となるフォルダを指定します。

### ステップ 2: スキャンの実行
1. ウィンドウ下部の **「スキャン」** ボタンをクリックします。
2. スキャンが完了すると、「結果」セクションに未参照のテクスチャ一覧と件数が表示されます。
3. 検出された各テクスチャの右側にある **「選択」** ボタンをクリックすると、Project ウィンドウ上で該当アセットがハイライト（Ping）され、実際にどのような画像か確認することができます。

### ステップ 3: 削除の実行
1. 未参照テクスチャが存在し、削除しても問題ないことを確認したら、ウィンドウ下部の **「削除」** ボタンをクリックします。
2. 確認ダイアログが表示されるので、**「実行」** をクリックすると削除処理（またはゴミ箱への移動）が行われます。

---

## 4. 注意事項

> [!WARNING] **参照範囲の制限について**
> このツールは、**「参照元フォルダ」で指定されたフォルダ配下のアセット** からの参照のみをチェックします。
> もし「参照元フォルダ」に指定していないフォルダ（例: 他のパッケージ内、または指定から漏れたフォルダ）のアセットが該当テクスチャを参照している場合でも、このツールは「未参照」と判定し、削除対象にしてしまいます。
> スキャン結果の削除を実行する前に、本当に他の場所で使用されていないか十分にご注意ください。
